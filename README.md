# Technical writing portfolio — Nicholas Best

A collection of documentation case studies across software, hardware, and scientific industries.

## Case Studies

### <a href="https://relishiq.com/relish-invoice-ai/" target="_blank">Relish Invoice AI and Data Assure</a>
**Context:** User and administrator manuals for invoice processing and bank data corroboration.  
**Authoring tool:** HubSpot.  
**Challenge:** Customer's information was incorrect, deprecated, mixed, duplicated, and inconsistent.  
**My contribution:** Restructured and expanded Relish's knowledge base by creating high-density articles, using GIFs and diagrams to clarify complex automated workflows and invoice validation states.

### <a href="https://www.kapsch.net/en" target="_blank">KAPSCH Operian OBO</a>
**Context:** Operator's manual for an automatic tolling system for vehicles in high-speed lanes. The system uses cameras at different locations in a road to detect vehicle type, speed, and the distance covered on a road trip. The system uses this information to automatically validate the driver's credentials and calculate a toll for the trip.  
**Authoring tool:** Confluence (exported to Word).  
**Challenge:** Document was unstructured, contained multiple mistakes, and styling issues.  
**My contribution:** Systematized fragmented documentation into a cohesive operator's manual, improving the clarity of vehicle detection workflows and toll calculation logic.

### <a href="https://www.otodatatankmonitors.com/" target="_blank">Otodata TM5240</a>
**Context:** Installation guide for the Digital Hall (DH) effect and Hydrostatic Pressure (HP) gas tank level monitors. The monitors are installed in vertical or horizontal gas tanks and use a sensor to monitor gas levels.  
**Authoring tool:** MadCap Flare.  
**Challenge:** Documentation was short and overly simplified.  
**My contribution:** Transformed simplified notes into a comprehensive installation guide, developing detailed topics for DH and HP sensor variants to reduce field-setup ambiguity.

### <a href="https://alcorscientific.com/clinical-lab/ised/" target="_blank">Alcor iSED Elite ESR Analyzer</a>
**Context:** Service and maintenance manual for an automated Erythrocyte Sedimentation Rate (ESR) blood analyzer. The device uses a robotic arm to place a blood sample test tube on a mixing wheel, analyzes the sample, and generates an ESR test result.  
**Authoring tool:** MadCap Flare.  
**Challenge:** Documentation was written and presented in Word, used old graphics, confusing diagrams with words or paragraphs inside the diagrams, and unstructured content.  
**My contribution:** Supported the modernization of legacy Word documentation by migrating content into MadCap Flare and cleaning up technical illustrations for better visual clarity.

### <a href="https://dantex.com/pico-inkjet-digital-presses/picojet-1200/" target="_blank">Dantex PicoJet 1200 UV Inkjet Hybrid Press</a>
**Context:** Operator's manual for a machine used to print and manufacture labels that have a wide range of uses and manufacturing requirements. A media film is fed into the machine, passing through multiple sections that clean, energize for adhesion, and treat the film with an anti-static bar before printing the labels. After printing, the media film is cured using a UV light and is cut into a desired size and shape.  
**Authoring tool:** MadCap Flare.  
**Challenge:** Document's safety information was presented on chapter 3, contained incorrect or deprecated information, styling was inconsistent.  
**My contribution:** Contributed to the update of the operator's manual, focusing on correcting deprecated safety information and ensuring consistent styling across technical chapters.

### <a href="https://code.kx.com/q/" target="_blank">KX Systems kdb+q</a>
**Context:** Documentation support for high-performance, column-oriented databases used in high-frequency trading and real-time analytics. Supported the technical reference for the q programming language and kdb+ database functions.  
**Authoring tool:** MadCap Flare.  
**Challenge:** Documentation was hard to follow, and the coding examples were inconsistent.  
**My contribution:** Improved the technical usability of q-language documentation by standardizing code examples and implementing a more logical migration into the Flare authoring environment.

### <a href="https://voltalis.co.uk/" target="_blank">Voltalis System</a>
**Context:** Installation guide for an Automated Demand Response (ADR) system. The system uses a communicator and a mobile app to control a household's electrical devices that are connected to the communicator.  
**Authoring tool:** MadCap Flare.  
**Challenge:** The only documents available were brochures, marketing PDFs, and internal documents. The client needed documentation for the UK market.  
**My contribution:** Authored a localized installation suite for the UK market from scratch, translating engineering requirements and marketing brochures into functional technical procedures.

### <a href="https://plasma.oxinst.com/products/icp-etching/plasmapro-100-cobra-icp" target="_blank">Oxford Instruments PlasmaPro 100 Cobra</a>
**Context:** Operator and service manual for a modular plasma processing system designed to etch silicon wafers and other substrates for solar panels.  
**Authoring tool:** Adobe FrameMaker.  
**Challenge:** 300+ page manual with highly complex terminology.  
**My contribution:** Managed the migration and technical editing of high-complexity semiconductor etching chapters, maintaining terminology precision for a 300+ page scientific manual.

### <a href="https://www.ultra-ic.com/ewst/rss8000/" target="_blank">Ultra I&C Galileo RSS8000/DF Threat Simulator</a>
**Context:** Maintenance manual for an Angle of Attack (AOA) radar threat simulator with multiple amplitude comparison Direction Finding (DF) outputs. The software is designed to test aircraft defense aids subsystems.  
**Authoring tool:** Adobe FrameMaker.  
**Challenge:** 600+ page manual with highly complex documentation and diagrams. Sensitive information.  
**My contribution:** Handled the migration of high-density radar threat simulation data, ensuring the integrity of complex signal-processing diagrams and sensitive maintenance procedures.

### <a href="https://www.usa.philips.com/healthcare/product/HC989706000051/tempusproruggedadvancedmonitor?srsltid=AfmBOorpq_BnpvwvS6oERkjZaX2ylUzNM_TTahwD5jE7lybda9Zwiq0e" target="_blank">Philips Tempus Pro Patient Monitor</a>
**Context:** Service manual for a paramedical device that provides Electrocardiogram (ECG) monitoring, Non-Invasive Blood Pressure (NIBP), Pulse Oximetry (SpO₂), and impedance respiration detection.  
**Authoring tool:** Adobe FrameMaker.  
**Challenge:** 300+ images with inconsistent markings or text.  
**My contribution:** Executed pixel-level cleanup of 300+ electrical illustrations to ensure visual consistency in high-stakes paramedical service environments, while co-authoring core monitoring topics.

### <a href="https://github.com/Nicholas-Best/Tesis.git" target="_blank">Personal Python Script: Automated Data Processing & Visualization</a>
**Context:** I had hundreds of raw data points in IRAF (Image Reduction and Analysis Facility). The manual process of combining multi-night data and performing linear adjustments was time-intensive and prone to human error.  
**Authoring tool:** Python, Jupyter Notebook, Pandas (for data manipulation), Matplotlib (for visualization).  
**Solution:** I created a Python-based workflow that:
  - Automated data cleaning and variable assignment across multiple observation nights.
  - Generated linear regression models and high-resolution graphs to visualize data trends.
